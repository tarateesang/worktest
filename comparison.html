We moved from a single solution for many different tasks, to cloud tools that do a single job very well and integrate with other services. That's exactly what Cogo aims to be. We are completely focused on enabling the best knowledge center functionality.

Cogo vs. Sharepoint

Here are a couple reasons why Cogo is a better fit for your needs.

Fast & easy every time

Sharepoint offers building blocks to build a knowledge center, but it requires a lot of setup and customization by developers, time and time again.

Cogo doesn't - it's fast and easy to set up every time.


Modernizing towards the cloud

Since we all use cloud apps that can easily integrate with each other, we do not need to use one ecosystem anymore.

Instead, Cogo lets you mix and match the best tools together.

All the features, none of the complexity

Sharepoint has almost any feature you can imagine. However, it is very complex and clunky. The customizability means that you almost always need to work with a developer to set it up for a particular use case; and even then, it's very complex to operate.

Cogo is designed to be used by an everyday user, meaning users can customize the platform however they want, but don't have to jump through hoops to use it.

Cogo vs. Sharepoint

Here are a couple reasons why Cogo is a better fit for your needs.

Better user experience for adding files or links

In Confluence, when you are adding files or links in confluence, you first have to create a 'page' and then within the page, you can add different types of content.

Cogo provides an intuitive, straightforward experiences for adding and creating new items. You can choose to drag & drop, or manually enter information for new items.

Powerful tagging system for categorization

Confluence uses "labels" and does not recommend tags, so you have to add the tags manually to each item.

Cogo not only recommends tags, automating the process, but also supports a powerful two-level tagging system to quickly drill down the right content.

Relevant search results & clear presentation

Confluence's search is much less powerful than Cogo's. Not only is it limited to keywords, but results are not recognizably displayed.

With Cogo, quality and popularity of information are also taken into account. Results are also presented with a recognizable cover and the ability to quickly preview content.

Presentations on the go

Unlike Cogo, Confluence does not have a presentation mode feature. This means that information is stored, but you must find another tool to present content in meetings, workshops, or client presentations.

Create collections

In Confluence, it is not possible to group items together. You are unable to create collections for workshops or even just selecting the best items for a certain topic. However, this isn't an issue with Cogo.

Lightweight and elegant

Like Sharepoint, Confluence is a very complex tool that can be used for many things (i.e. project management, cloud storage, knowledge center). However, it lacks lightweightness and interoperability with other modern collaboration and communication tools.

Cogo vs. Box.com

Here are a couple reasons why Cogo is a better fit for your needs.

Tagging capabilities

With Box.com, you are unable to tag content for different industries or domains in Box.com.


Another way to organize and sort items

Box.com relies on the use of folders, so items would translate to folders containing ...

With Cogo, an item is a cohesive concept and people do not have to deal with navigating folders and opening files.


Increased discoverability of information

With Box.com (or other cloud storage solutions), you need to make sure that everyone is organizing and naming content the same way. This means that everyone has to use the same naming and organization conventions.

Due to Cogo's tags and search feature, information can be shared easily without the disorganization and difficulty navigating.

Deep folder hierarchies

In Box.com, a knowledge center consists of files and folders. When you open the repository, you must process lists of files and folders, without any recognizable visual imagery.

Cogo creates and uses recognizable covers, sorted by relevance with shortcuts, to automatically preview primary content.


A front-end integration solution

Typically, if you are using Box.com (or any other cloud storage solution), you need to use the storage solution with a fully-featured, front-end service like Cogo.

Currently, are finalizing integrations for most cloud storage solutions with Cogo, allowing you to make powerful combinations for your needs.
